Learning-based model predictive control for autonomous racing

Juraj Kabzan et al., IEEE RAL 2019

  • Main idea: Having a nominal model from physics, then use Gaussian Process to estimate the model error and niose. Handcraft cost function for the MPC
  • Apply gaussian progress (GP) regression to collected data to approach the deviation between the nominal model and the real one.
    A GP is yk=dtrue(zk)+wk, dtrue is the real, unknow function.
    wk∼N(0,∑w) is Gaussian noise with diagonal variance ∑w=diag([σ12,⋯,σnd2])
    d(z)∼N(ud(z),∑d(z)), model error
    Use sparse GP regression to reduce the computational complexity.
  • Using a dynamic bicycle model f to estimate a racing car, assuming that there is a deviation between the simple model and the real model
    • xk+1=f(xk,uk)+yk, where yk=Bd(d(zk)+wk)
      d additional learned part of the dynamics, estimating the model error dtrue, only affect part of the model, determined by Bd
    • Bd is determined by hand (assuming the model error only affects the dynamic part of the system)
    • real_model = nominal_model + model_error + gaussian_noise
  • The model can be learned by:
    yk=Bd†(xk+1−f(xk,uk))=dtrue(zk)+wk
  • The cost function for MPC contains three parts:
    1. contouring cost
    2. (regularization) quadratic cost on the steering angle
    3. (regularization) aggressiveness
  • Track, tire and input constraints
